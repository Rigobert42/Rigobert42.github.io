# Créer des outils et applications informatiques pour les R&T

## Présentation
Cette compétence vise le développement d’outils utiles aux réseaux et télécommunications : scripts, applications web, interfaces d’administration, automatisation ou exploitation de données. Elle relie les besoins réseau à la programmation, à la gestion de données et à la maintenance logicielle. Dans un portfolio ROM, elle permet de montrer que tu sais aussi produire des outils qui facilitent l’exploitation et l’automatisation [file:1][file:3].

## Apprentissages critiques
| Niveau | Apprentissages critiques |
|---|---|
| Novice | Utiliser un système informatique et ses outils ; lire, exécuter, corriger et modifier un programme ; traduire un algorithme dans un langage donné ; connaître l’architecture d’un site web ; choisir les mécanismes de gestion de données adaptés ; s’intégrer dans un environnement propice au développement et au travail collaboratif. |
| Intermédiaire | Automatiser l’administration système avec des scripts ; développer une application à partir d’un cahier des charges ; utiliser un protocole réseau pour programmer une application client/serveur ; installer et administrer un système de gestion de données ; accéder à un ensemble de données depuis une application ou un site web. |
| Compétent | Élaborer les spécifications techniques et le cahier des charges d’une application ; mettre en place un environnement de travail collaboratif ; participer à la formation des utilisateurs ; déployer et maintenir une solution informatique ; s’informer sur les évolutions technologiques ; sécuriser l’environnement numérique d’une application. |

## Lecture pour le portfolio
Cette partie du portfolio peut accueillir un projet de script, d’outil de supervision, de mini-application web ou de gestion de base de données. Le plus important est de faire ressortir l’objectif fonctionnel, les contraintes techniques et la valeur ajoutée de ton outil pour l’équipe ou l’infrastructure. Les preuves les plus convaincantes sont le code, les captures d’interface, les tests, la documentation et les explications de conception [file:1][file:3].

## Analyse réflexive
Cette compétence m’a appris à ne pas voir le développement comme une fin en soi, mais comme une réponse à un besoin précis. J’ai progressé dans la structuration du code, dans l’usage d’outils collaboratifs et dans la rédaction de documentation utile à d’autres. J’ai aussi compris qu’un petit outil bien conçu peut avoir un vrai impact sur l’efficacité d’une équipe technique.
